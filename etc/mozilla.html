Mozillaの不具合メモ

私は、基本的にMS Windows上でMozillaを 常用のウェブブラウザとして使っています。 他の人にもMozillaを奨める時に 「『×××』の機能が無いことだけ我慢すれば、十分に実用だよ！」 と言っています。 しかし、『×××』にどんなものがあったのか、忘れがちです。 というわけで、そのためのメモです。

注意事項: なお、大半の不具合は Bugzillaに登録され、 バグ処理を待っている段階にありますので、 ひょっとすると、最新版のMozillaでは直っている機能もあるかも知れません。 ご了承下さい。

バグ一覧

印刷
印刷のデフォルト設定（ページサイズなど）が反映されない。 特に、起動するたびに、 ページサイズがLetterサイズになってしまうのが不便です。 （Mozilla 0.9.8 頃からの現象のようです）
cf. [bugzilla-jp 1011], [bugzilla 123554]
FTP
プロキシを通さない直接接続で FTP した時に、 ファイルの一覧などが取得できないみたいです。 プロキシ経由の接続を普段から使いましょう。
[bugzilla XXX]: バグ報告例を発見できませんでした。 （ご存じの方、教えてください）
学内のファイアウォールの影響である可能性が出てきました。 現在、IPCに問い合わせ中です。 (2001-10-23追記)
Java Plug-inがうまくインストールされない。
JDKをインストールしておけば、 インストーラがそれを自動認識してJava Plug-inを使うようになるはずですが、 なぜか手元ではできない場合がありました。 (再現条件は不明)
一時的な解決策: インストール時にJDKをインストールしたディレクトリのbin/NP*.dllを Mozilla のplugins/ディレクトリにほうりこんでおく。
cf. Mozilla 0.9.8 Release Notes: Java

解決したバグ

ULIS OPAC検索
いつからできるようなったのかわかりませんが…、2002-01-20現在、Mozilla 0.9.7で検索できるようになっていることを確認しました。
付属図書館のOPACの検索ができません。 MozillaとIEなどでは微妙にJavaScriptの実装が異なるので、検索できません。
対策: みんなで図書館に文句を言いましょう!! あと、Webcatもありますね。
印刷プレビュー
Mozilla 0.9.6で実装されました。（一部、不安定な部分も残っていますが…）
印刷結果をプレビューすることができません。 印刷のページ数があらかじめ分からないと不安なときには、困るでしょう。
参考: [bugzilla 20943]
Proxyの自動設定
Mozilla 0.9.4で使えることを確認しました。
JavaScriptを使ったProxyの自動設定機能がうまく働いていないようです。 直接プロキシサーバのホスト名・ポート番号を指定しましょう。
cf. [bugzilla 54804]
