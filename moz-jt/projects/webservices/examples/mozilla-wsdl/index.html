 Mozilla WSDL の動作例
Mozilla WSDL Example
Doron Rosenberg (doron@netscape.com)

Mozilla 1.4 β版で Gecko は WSDL に対応し、Web サービスと容易にやりとりできるようになりました。 ここの動作例では、ユーザが入力した文字列にマッチする書籍を探すために Amazon Web サービスに検索質問を投げます。
With Mozilla 1.4beta, Gecko has WSDL (Web Service Description Language) capabilities that allows it to easily communicate with web services.  In this example, the Amazon web services will be queried for books matching the string the user entered.

セキュリティに関する注意
Note about Security
 本動作例では、（mozilla.org から amazon.com へ）別のドメインに対して呼び出しを行う必要があるため、Mozilla のドメイン間セキュリティモデルに違反します。 Mozilla 1.4 β版には、特に Web サービス用に開発された新しいセキュリティモデルがあります。 これを使えば、Web サービスを呼び出す権限をブラウザに与えることができるようになります。 しかし、これは非常に新しい機能なので、現在公開されている Web サービスにはまだ対応していません。 従って、代わりに以下のユーザ設定を行うことにより、このセキュリティの制限を回避しなければなりません。
Since this example needs to call another domain (mozilla.org to amazon.com), it breaks the cross-domain security model that Mozilla has.  There is a new model developed especially for web services in Mozilla 1.4beta that allows a web service to grant the browser permission to call it.  However, since it is very new, it hasn't reached yet the public web services out there.  Therefore, as a workaround, you have to bypass it by setting the following user preference:
user_pref("signed.applets.codebase_principal_support", true);

 この設定により、Mozilla はユーザに amazon.com にアクセスするのを許可するかどうかを確認するようになります。
This will make Mozilla prompt you to allow it to access amazon.com.
動作例
Example
 動作例では、ユーザが Amazon Web サービスに与えたキーワードにマッチする書籍を検索できます。 例えば、"Mozilla" と打ち込んだ場合、この悪名高いトカゲ [ 訳注: Mozilla のこと ] に関連する本がヒットするでしょう。 この動作例を実行するにはここをクリックしてください。
The example allows the user to search the Amazon web service for books matching a given keyword.  For example, typing in "Mozilla" will match books related to the infamous lizard.  To run the example, click here.
どのように動作するのか
How it Works
 まず、Amazon 側の WSDL ファイルを使って "Amazon" オブジェクトを初期化します。 これにより、ブラウザでは Amazon Web サービス用のプロキシオブジェクトが生成され、Amazon のサービスがまるでローカルのリソースであるかのようにスクリプトから扱えるようになります。 WSDL ファイルにはインタフェースが記述されています。 これには、どのようなメソッドが呼び出せるか、どんな引数を渡すのか、どんな返り値が返ってくるのかなどが詳細に記述されています。 プロキシオブジェクトに対してあるメソッドが呼び出されると、Mozilla はこの WSDL ファイルに基づいた SOAP エンベロープを生成し、それを Amazon に送信します。 Amazon が返答を返すと、Mozilla は SOAP レスポンスを JavaScript のオブジェクトに変換します。 今回の場合は、タイトル・著者名の配列・定価・Amazon 価格などのような各書籍についての情報を含む "book" オブジェクトの配列が対象となります。
First, we instantiate an "Amazon" object using their WSDL file.  This creates a proxy object in the browser for the Amazon web service which can then be scripted as if it where a local resource.  The WSDL file describes the interface, detailing which  methods can be called, which arguments to pass in and what the response will look like.  When a method is called on the proxy object, Mozilla generates a SOAP envelope based on the WSDL file and sends it to Amazon.  When Amazon responds, Mozilla is able to convert that SOAP response into a JavaScript object.  In this case, an array of "book" objects, where each book contains information such as the title, an array of the authors, the retail and amazon's price and more. 

動作例では、この情報を受け取って、それをそのページの DOM に追加することで表示を行います（新たにページを読み込むわけではありません）。
The example then takes this information and displays it by adding it to the page's DOM - no new page has to be loaded.
