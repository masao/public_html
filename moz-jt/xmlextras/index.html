XML 追加モジュール

XML Extras

ニュースグループ: netscape.public.mozilla.xml
ページ管理者: Heikki Toivonen

Newsgroup: netscape.public.mozilla.xml
Page maintained by: Heikki Toivonen

XML 追加モジュールは、開発者に XML をデータ（つまりただのドキュメントフォーマットとしてでなく）として扱うことを可能とするいくつかの機能を含んでいます。 モジュールは drop-in のコンポーネントとして構成され、その XML-as-data（XML をデータとして扱う）の機能を JavaScript と C++/XPCOM ユーザー両方に公開します。 XML 追加モジュールは標準で全プラットフォームでビルドされ、ブラウザ・インストーラに含まれていますので、Nightly ビルドでも利用できます。

The XML Extras module contains several features that allow developers to treat XML as data i.e. not as just another document format. The module is structured as a drop-in component and exposes its XML-as-data features both to JavaScript and C++/XPCOM users. The XML Extras module is built by default on all platforms, and is included in the browser installers so it is available in the nightly builds.

品質保証とテスト

QA and Testing

主に XMLHttpRequest を通じて HTTP の GET メソッドを行うオンラインのテストもいくつかあります。

There are some online tests for mainly exercising the HTTP GET method via XMLHttpRequest.

上記で挙げた実装済のコンポーネントそれぞれについての他のテストケースは mozilla/extensions/xmlextras/tests/ にあります。 しかし、より網羅的なテストスイートを作る必要があります。 品質保証を手伝って頂ける方は、以下からお願いします。
Other test cases for each of the implemented components listed above exist in mozilla/extensions/xmlextras/tests/. More exhaustive test suites need to be created, however. If you'd like to help out with QA, you can do one of the following:

  • 実装済のコンポーネントどれかに品質保証オーナーとして参加してください。 品質保証オーナーは、それらのコンポーネントに付随したテストを管理し、 関連するバグに対する品質保証の窓口として活動し、 回帰テストを手伝います。
  • Sign up as the QA owner for any of the implemented components. The QA owner will manage the tests associated with their component, act as QA contact for related bugs and help with regression testing.
  • 実装済コンポーネントが使うテストスイートや個別テストを提供してください。 テストは品質保証オーナー（デフォルトオーナーは heikki@netscape.com ）に送るか、mozilla/extensions/xmlextras/tests/ にチェックインしてください。
  • Contribute a test suite or individual tests that use the implemented components. Tests can be sent to the QA owner (heikki@netscape.com is the default owner) and/or checked into mozilla/extensions/xmlextras/tests/.
  • 実装済コンポーネントに対するバグを提出してください。 バグは、Browser プロダクトの XML コンポーネントに提出してください。
  • File bugs against the implemented components. Bugs should be filed against the XML component in the Browser product.

文書化

Documentation

たぶん、ここで述べている技術の利用方法を学ぶ最も良い方法は、 動作例によるものでしょう。 mozilla/extensions/xmlextras/tests/ にいくつかの動作例があります。 動作例を見るには上記のリンクをたどって、 HTML 文書のページソースを見る必要があります。

Probably the best way to learn how to use these technologies is through examples. There are some in mozilla/extensions/xmlextras/tests/. If you follow that link to look at the examples you will need to look at page source for the HTML documents.

XMLHttpRequest オブジェクトでは、Microsoft XMLHttpRequest ドキュメント （英語） にほぼ準拠しています。 ただし、以下の点に注意すること。 全ての関数名とプロパティ名は小文字で始まります。 オブジェクト生成の方法が異なります。 いくつかのプロパティは実装されていません。

For XMLHttpRequest object you can mostly rely on the Microsoft XMLHttpRequest documentation, with some caveats: all functions and property names begin with a lower case letter and the object creation is different. Some properties are not implemented.

Thad Hoffman は、Mozilla で XML Data Islands を真似る方法を文書化しました。 Edmond Woychowsky も Mozilla での XML Data Islands についての記事: "XML data islands work in Mozilla" （英語） と "Implement a flexible shopping cart with XML and ASP" （英語） を書いています。

Thad Hoffman has written a document that shows how you can mimic XML Data Islands in Mozilla. Edmond Woychowsky has also written articles on XML Data Islands in Mozilla: "Make XML data islands work in Mozilla" and "Implement a flexible shopping cart with XML and ASP"
